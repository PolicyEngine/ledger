Arch Fact identity v2 architecture
Diagram Example Keys Rules Lineage

Source facts stay stable. Targets stay downstream.

Arch records publisher observations with provenance, source-cell lineage, dimensions, constraints, concept alignment, and reproducible build hashes. Microplex consumes semantic handles later through a thin adapter.

Principles

Keys come from schema fields, not labels or table layout.

Lineage proves where a fact came from, but does not define the fact.

Concept alignment is reviewable metadata, not source fact identity.

Microplex owns source selection, aging, reconciliation, and activation.

End-to-end shape

The primary Arch key is source-specific. The semantic key is the source-agnostic bridge that downstream systems can reconcile and select from.

Publisher publications

Workbooks, CSVs, PDFs, APIs, and official release pages from IRS, Census, SSA, CBO, BEA, CMS, and other primary publishers.

Object storage mirror

Raw artifacts are immutable, content-addressed, and versioned. R2 or another object store is storage metadata, not fact identity.

content_hash + publisher_url + retrieval metadata

SourceRelease

The immutable publisher release or revision. If bytes change under the same vintage, Arch creates a new release revision.

source_release_key

SourceArtifact

A logical artifact inside the release. File names and URLs are audit metadata.

source_artifact_key

SourceFrame

Sheet, table, section, frame, or extracted table family with parser and layout metadata.

source_frame_key

SourceRecord

Native row, section, record, or synthetic legacy record inside the immutable source release.

source_record_key

SourceCell / SourceField

Value cells and field lineage for the value, measure, period, geography, dimensions, filters, and constraints.

source_cell_key

ObservedMeasure

The source-native quantity: source measure ID, unit, scale, quantity kind, and measurement basis.

observed_measure_key

AggregateFact

One source-specific observation with value payload, dimensions, constraints, hashes, and lineage references.

aggregate_fact_key

Build hashes

Values are not identity. Drift is detected with value, lineage, content, artifact, and build hashes.

value_hash + lineage_hash + content_hash

Validation

Duplicate aggregate keys with different values fail. That points to missing dimensions or source ambiguity.

ConceptAlignment

Evidence-bearing mapping from source measure to canonical concept. Review notes can change without rekeying source facts.

concept_alignment_key

SemanticFact

Source-agnostic statistic for comparing equivalent facts across publishers, releases, and packages.

semantic_fact_key

Fact registry

Queryable SQLite, Parquet, JSONL, or Supabase tables with the same Arch schema and validation contract.

adapter boundary

Microplex adapter

Thin downstream mapping from Arch facts into Microplex constraint and value tables. No Microplex imports in Arch core.

Microplex Targets

Owns source selection, aging, reconciliation, target activation, simulator aliases, and target set composition.

Calibrated microdata

Final jurisdiction-specific model inputs and calibrated outputs are downstream products, not Arch source facts.

What does not key the aggregate: the label "Adjusted gross income", the cell address TBL11!D10, the evidence URL for IRC section 62, the current parser version, the R2 mirror path, or any Microplex variable alias. Those remain lineage, evidence, build, storage, or adapter metadata.

Aggregate key material

  • Source: release and reporting series.
  • Measure: observed source measure, unit, scale, and basis.
  • Coordinates: period, geography, entity, aggregation.
  • Meaning: dimensions and universe constraints.

Never key on

  • Display: labels, notes, row order, cell address, sheet layout.
  • Review: evidence notes, evidence URLs, alignment status.
  • Build: parser version, package commit, build timestamp.
  • Downstream: Microplex target name, source selection, aging, aliases.

Agent production bar

  • L0: synthetic legacy facts stay non-production.
  • L1: artifact-only facts are drafts.
  • L2: value cell lineage is the production minimum.
  • L3: full cell lineage covers values, dimensions, filters, constraints.
